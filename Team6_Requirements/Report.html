Project 1 Overview:

In this project, we got a basic understanding of the ChocAn system, created a glossary of terms and a use-case diagram, and wrote use-case descriptions. We had to assume that were responsible to write only the ChocAn data processing software and that another organization will be responsible for the communications software, for designing the ChocAn provider’s terminal, for the software needed by Acme Accounting Services, and for implementing the EFT component. As a result of this, we assumed that our system is only responsible for allowing providers to bill ChocAn and request provider directories, allowing operators to modify statuses, allowing managers to access reports, and running the weekly accounting procedures. Additionally, we familiarized ourselves with HTML, Eclipse, UMLetino, and Bitbucket. We each created our own HTML files with header and task distribution tables, as well as our own individual use-case diagrams, which are also in our submission folder. The glossary, group use-case diagram, and use-case descriptions are on this Report.html file. The header and task distribution tables are also on this Report.html file along with the Bitbucket statistics image. The submitter of the Report.html file is Luca Hallenbeck.

Use Case Diagram:

Glossary:

  • Acme Accounting Services – Third-party group responsible for financial procedures
  • ChocAn Data Center – Where the main accounting procedure is run; houses info about members/providers and their status
  • Chocoholics Anonymous (ChocAn) – An organization dedicated to helping people addicted to chocolate
  • EFT Data – Electronic funds transfer data used to pay providers
  • Manager – Individual that receives summary reports
  • Member – A member of the ChocAn organization
  • Member Card – Plastic card embossed with the member’s name and a nine-digit member number and incorporating a magnetic strip on which that information is encoded
  • Member Report – Weekly report received by each member who consulted a ChocAn provider during previous week; lists services provided to that member sorted in order of service date
  • Operator – Individual with the ability to add/delete/update members and providers of ChocAn; Works at ChocAn data center
  • Provider – Health care professional that provides services to ChocAn members
  • Provider Computer Terminal – specially designed ChocAn computer terminal, similar to a credit card device in a shop, that reads the member card, dials the ChocAn data center, and verifies member number, as well as display various data and forms
  • Provider Directory – Ordered list of services and corresponding codes and costs
  • Provider Report – List of services provided to ChocAn members
  • Service Code – Six-digit code corresponding to a service provided by one of the providers; found in provider directory
  • Summary Report – Lists every provider to be paid that week, the number of consultations each had, and their total fee; sent as an e-mail attachment
  • Timer – Automatically completes procedures and reports

Use Case Description List:

Use Case: Run weekly accounting procedures
Context: The week's files of services provided are read and several reports are printed.
Actors: Timer
Main Success Scenario:

1. Timer recognizes that it is midnight on Friday
2. System is alerted to begin running weekly accounting procedures
3. File containing all of the service provided by health care professionals is read
4. System automatically begins compiling
 4.1 System generates summary report
 4.2 System generates service report
 4.3 System generates member report
5. System sends out reports to consumers and providers as email attachments

Extensions: None



Use Case: Bill ChocAn
Context: Provider acquires appropriate account and service information from member to bill ChocAn for said service(s).
Actors: Provider
Main Success Scenario:

1. Provider swipes member card or keys in member number to verify membership
 1.1 If number is valid, “Validated” is displayed on the screen
2. Key in date of service in MM-DD-YYYY format
3. Search Provider Directory for appropriate six-digit service code
 3.1 Key in service code
4. Verify that service was provided
5. Enter comments about service provided (optional)
6. Return to provider menu

Extensions:

1.1a. If number is invalid, “Invalid number” is displayed on the screen
1.1b. If member owes fees, member status is suspended and “Member Suspended” is displayed on the screen
4a. Software prints error message if service code doesn’t exist



Use Case: Access Report
Context: A ChocAn manager can request to run any report individually from the week’s file of services at any time during the week.
Actors: Manager
Main Success Scenario:

1. Manager decides a report is necessary
2. Manager prompts system to run a report
3. Report is compiled by system and printed for viewing by manager

Extensions:

2a. Manager prompts system to run a service report
2b. Manager prompts system to run a member report
2c. Manager prompts system to run a summary report



Use Case: Modify Status
Context: During the day, the ChocAn Data Center runs in interactive mode to allow the operator to edit member and provider information.
Actors: Operator
Main Success Scenario:

1. Choose to edit member information
2. Choose to edit provider information
3. Return to operator menu

Extensions:

1a. Delete member from the ChocAn system if they resigned
1b. Update member records
1c. Add member to ChocAn system
2a. Delete provider from ChocAn system
2b. Update provider records
2c. Add provider to ChocAn system


Use Case: Request provider directory

Actors: Provider
Main Success Scenario:
