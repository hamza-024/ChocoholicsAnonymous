Project 1

Project 1 Overview:

In this project, we utilized UMLentino, Bitbucket, HTML, and Eclipse to collaborate with each other and create the solution. We were asked to create data processing software for the company Chocoholics Anonymous. First, we had to read through the project description to understand our objectives and seek out key terms. We then turned these key terms into a glossary, defining what may need to be defined for viewers of this project. Second, we each made our own use case diagrams to break down the functions of this software system. We combined our individual use case diagrams into one group diagram. Finally, we used the diagram to create a list of use cases and break down these functions further. We provided the name of the use case, its context, what actors participated in it, what was done step by step, and what possible extensions of each step were. These three components were formatted into an HTML file (both for the individual and for the group), along with a task distribution table and a Bitbucket statistics picture for the group file.

Glossary:

  • Acme Accounting Services: third-party organization responsible for financial procedures and processing payments of ChocAn membership fees
  • Chocoholics Anonymous (ChocAn): an organization dedicated to helping people addicted to chocolate
  • ChocAn Data Center: contains information on ChocAnâ€™s members and providers; runs accounting procedures
  • EFT: electronic funds transfer
  • Operator: employees that utilize the ChocAn Data Center to edit member and provider information
  • Provider: health care professional that provides services to ChocAn members
  • Provider Directory: alphabetically ordered list of service names and corresponding service codes and fees
  • Software Product: the data processing software created by our organization for ChocAn's system

Use Case List:

Use Case: Request Provider Directory
Context: The provider can request to see the full Provider Directory at any point.
Actors: Provider
Main Success Scenario:

1. Request Provider Directory from software product
2. Receive email-attachment of Provider Directory

Extensions: None



Use Case: Bill ChocAn
Context: Provider acquires appropriate account and service information from member to bill ChocAn for said service(s).
Actors: Provider, ChocAn Data Center
Main Success Scenario:

1. Swipe member card or key in member number
2. Key in date of service in MM-DD-YYYY form
3. Enter service code
 3.1 Search Provider Directory for appropriate six-digit service code
 3.2 Add comments about service provided (optional)
4.Write record to disk

Extensions:

3a. Print error: service code does not exist



Use Case: Edit member information
Context: Operator changes member information in system if needed.
Actors: Operator
Main Success Scenario:

1. Member information has changed
2. Operator chooses to update information in system

Extensions:

2a. Add member
2b. Remove member if they have resigned
2c. Update member records



Use Case: Edit provider information
Context: Operator changes provider information in system if needed.
Actors: Operator
Main Success Scenario:

1. Provider information has changed
2. Operator chooses to update information in system

Extensions:

2a. Add provider
2b. Remove provider
2c. Update provider records


Use Case: Send weekly service reports
Context: Look through services provided during the week and create reports to send to appropriate recipients.
Actors: ChocAn Data Center
Main Success Scenario:

1. Weekly file of services is read
2. Reports are printed and sent out
  2.1 Send email to each member who has received a service that week, including their personal information, as well as information on the service(s)
  2.2 Send email to each provider who has billed ChocAn that week, including the list of services they provided to members

Extensions: None


Use Case: Write EFT data to disk
Context: Record EFT data and check provider bank accounts.
Actors: ChocAn Data Center
Main Success Scenario:

1. Create record of EFT data for payments that week
2. Write record to a disk
3. Banking computers will ensure that providers received appropriate payment

Extensions: None


Use Case: Record payments
Context: Acme Accounting Services takes member payment information from ChocAn to process and document.
Actors: Acme Accounting Services
Main Success Scenario:

1. Check ChocAn system to see if members have paid their fees
2. Record payments into Acme system

Extensions: None


Use Case: Suspend member
Context: Acme Accounting Services determines that a member needs to be suspended from ChocAn system.
Actors: Acme Accounting Services
Main Success Scenario:

1. Acme checks payment records to see that member has not paid fees
2. Member gets suspended from ChoAnsystem

Extensions: None


Use Case: Reinstate member
Context: Acme Accounting Services determines that a member can be reinstated into the ChocAn system.
Actors: Acme Accounting Services
Main Success Scenario:

1. Acme checks payment records to see that suspended member has paid overdue fees
2. Suspended member gets reinstated with ChocAn

Extensions: None
